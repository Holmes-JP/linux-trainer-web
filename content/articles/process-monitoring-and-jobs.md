## 「原理」から「実務」へ

[プロセスとシグナル](/articles/processes-and-signals) では PID・親子関係・シグナルという **仕組み** を扱った。ここではその仕組みを使って、**動いているものを監視し、名前で捕まえ、優先度まで調整する** 実務側をまとめる。

## ps と top — 一覧とリアルタイム

`ps aux` はその瞬間の全プロセスを **一度だけ** 表示する。「今どうなっているか」の静止画だ。

```
$ ps aux
USER   PID %CPU %MEM COMMAND
user   214 98.5  0.5 /usr/local/bin/batch.sh
user   240  0.0  0.1 -bash
```

`top` は同じ情報を **数秒ごとに更新** し続ける動画版で、CPU を食っているプロセスが上に並ぶ。暴走を「今まさに」追いたいときに使い、`q` で抜ける。

- 一覧を grep したり PID を控えたい → `ps`
- 何が重いのかを眺めて張り込む → `top`

## free と uptime — 負荷とメモリ

「重い」の正体は CPU 負荷かメモリ不足のどちらかが多い。それぞれ専用の見方がある。

```
$ uptime
 10:00:00 up 3 days,  2:11,  load average: 4.52, 3.98, 2.10
$ free -h
              total        used        free
Mem:           2.0Gi       1.8Gi       120Mi
```

`uptime` の **load average** は「実行待ちの行列の長さ」を 1分・5分・15分平均で示す。CPU コア数を超え続けていたら過負荷。`free -h` は空きメモリで、`free` がほぼ尽きているとスワップやプロセス強制終了の原因になる。`-h` は人間に読みやすい単位 (Gi/Mi) にするオプション。

## pgrep / pkill — 名前で探して止める

PID を毎回 `ps | grep` で探すのは面倒だ。名前で直接操作できるのが `pgrep` と `pkill`。

| コマンド | 意味 | 例 |
|---|---|---|
| `pgrep -a batch` | 名前に一致するプロセスの PID を表示 (`-a` でコマンド行も) | 何が動いているか確認 |
| `pkill batch` | 名前に一致するプロセスへ TERM を送る | まとめて止める |
| `pkill -9 batch` | 同じ対象へ KILL を送る (強制終了) | TERM で死なないとき |

`pkill` は **一致した全プロセスに一斉に効く** ので、名前が短すぎると巻き添えが出る。実行前に必ず `pgrep -a` で「何が当たるか」を確認してから撃つのが安全な作法だ。

## ジョブ制御 — 1 枚のターミナルで複数を回す

この演習はターミナルが1枚。長い処理を前面 (フォアグラウンド) で動かすと、その間コマンドを打てなくなる。**ジョブ制御** はこれを解決する。

| 操作 | 意味 |
|---|---|
| `コマンド &` | 最初からバックグラウンドで起動する |
| `Ctrl+Z` | 実行中のジョブを一時停止する |
| `jobs` | 今のシェルが抱えるジョブを番号付きで一覧 |
| `bg %1` | ジョブ1を停止状態からバックグラウンドで再開 |
| `fg %1` | ジョブ1を前面に戻す |

```
$ sleep 300 &
[1] 517
$ jobs
[1]+  Running                 sleep 300 &
$ fg %1
sleep 300
```

ただしジョブはそのシェルに紐づくため、**シェルを閉じると子プロセスも一緒に終わる**。閉じても動かし続けたいときは `nohup コマンド &` で起動する (切断シグナル HUP を無視させる)。

## nice / renice — 優先度を譲る

重い処理を動かしたいが、他の作業の邪魔はしたくない。そんなときは **優先度 (nice 値)** を下げる。nice 値は `-20`(最優先)〜`19`(最も控えめ) で、数字が大きいほど「他に譲る」。

- 起動時に下げる: `nice -n 10 ./heavy.sh`
- 動作中のものを下げる: `renice 10 -p 517`

値を **下げる** (優先度を上げる) のは root だけができる。一般ユーザーは「譲る」方向にしか変えられない、と覚えておくと権限エラーで悩まない。

## トラブル対応の型

1. `top` か `uptime` / `free -h` で「CPU が重いのか、メモリが無いのか」を切り分ける
2. `pgrep -a 名前` で犯人の PID とコマンド行を特定する
3. 止めてよいなら `pkill 名前` (TERM)、死ななければ `pkill -9 名前` (KILL)
4. 止めたくないが重いだけなら `renice` で優先度を下げて共存させる
5. 自分が回している長い処理は `&` / `nohup` でバックグラウンド化し、ターミナルを空けておく
